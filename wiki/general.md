Notions
-------

Not everything should be made generic.

The choice of tech stack is as much part of the solution than the code, no need to over abstract it.

Contribute to existing OSS solutions instead of creating yet another one. Fork if necessary.

No code is untouchable, code churning should be the norm. Untouched code smells.

Don't blindly follow trends, patterns or vague notions like configurability, scalability, etc. Think about what you need.

Write code that's easy to delete, not easy to extend. Requirements will change.
